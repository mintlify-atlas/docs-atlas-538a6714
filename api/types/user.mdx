## Overview

This page documents all user-related TypeScript types used in the Zongolica application.

## UserProfile

User profile stored in the `user_profiles` table.

```typescript
interface UserProfile {
  id: string;
  email: string;
  full_name: string;
  avatar_url: string;
  provider: string;
  onboarding_completed: boolean;
  created_at: string;
  updated_at: string;
}
```

### Field Descriptions

<ParamField path="id" type="string">
  UUID - Primary key, references auth.users(id)
</ParamField>

<ParamField path="email" type="string">
  User's email address from authentication provider
</ParamField>

<ParamField path="full_name" type="string">
  User's full name
</ParamField>

<ParamField path="avatar_url" type="string">
  URL to user's profile picture (from OAuth provider)
</ParamField>

<ParamField path="provider" type="string">
  Authentication provider ("google", "facebook", "email")
</ParamField>

<ParamField path="onboarding_completed" type="boolean">
  Whether user has completed the onboarding flow
</ParamField>

<ParamField path="created_at" type="string">
  ISO 8601 timestamp of profile creation
</ParamField>

<ParamField path="updated_at" type="string">
  ISO 8601 timestamp of last profile update
</ParamField>

### Example

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "email": "juan@example.com",
  "full_name": "Juan Pérez",
  "avatar_url": "https://lh3.googleusercontent.com/a/...",
  "provider": "google",
  "onboarding_completed": true,
  "created_at": "2024-03-15T10:30:00.000Z",
  "updated_at": "2024-03-15T10:30:00.000Z"
}
```

## UserPreferences

User preferences collected during onboarding, stored in the `user_preferences` table.

```typescript
interface UserPreferences {
  id?: string;
  user_id: string;
  experiencia: string[];
  duracion: string;
  dificultad: string;
  grupo: string;
  intereses: string[];
  created_at?: string;
}
```

### Field Descriptions

<ParamField path="id" type="string" optional>
  UUID - Primary key (auto-generated)
</ParamField>

<ParamField path="user_id" type="string">
  UUID - References user_profiles(id)
</ParamField>

<ParamField path="experiencia" type="string[]">
  Types of experiences the user prefers
  
  **Possible values:**
  - "aventura" - Adventure activities
  - "naturaleza" - Nature and landscapes
  - "cultura" - Cultural experiences
  - "gastronomia" - Food and beverages
  - "relax" - Relaxation and wellness
</ParamField>

<ParamField path="duracion" type="string">
  Preferred duration of visits
  
  **Possible values:**
  - "medio-dia" - Half day (2-4 hours)
  - "dia-completo" - Full day (6-8 hours)
  - "fin-de-semana" - Weekend (2-3 days)
</ParamField>

<ParamField path="dificultad" type="string">
  Physical difficulty level the user is comfortable with
  
  **Possible values:**
  - "facil" - Easy (accessible to all)
  - "moderado" - Moderate (requires some fitness)
  - "dificil" - Difficult (requires good physical condition)
</ParamField>

<ParamField path="grupo" type="string">
  Who the user typically travels with
  
  **Possible values:**
  - "solo" - Solo traveler
  - "pareja" - Couple
  - "familia" - Family with children
  - "amigos" - Friends
</ParamField>

<ParamField path="intereses" type="string[]">
  Specific interests and attractions
  
  **Possible values:**
  - "cascadas" - Waterfalls
  - "miradores" - Viewpoints
  - "cuevas" - Caves
  - "senderismo" - Hiking
  - "rappel" - Rappelling
  - "kayak" - Kayaking
  - "historia" - History
  - "fotografia" - Photography
</ParamField>

<ParamField path="created_at" type="string" optional>
  ISO 8601 timestamp (auto-generated)
</ParamField>

### Example

```json
{
  "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "experiencia": ["aventura", "naturaleza"],
  "duracion": "dia-completo",
  "dificultad": "moderado",
  "grupo": "familia",
  "intereses": ["cascadas", "miradores", "senderismo"],
  "created_at": "2024-03-15T10:35:00.000Z"
}
```

## Authentication Functions

Functions for user authentication.

### signInWithGoogle()

Initiate Google OAuth sign-in flow.

```typescript
function signInWithGoogle(redirectPath?: string): Promise<{
  data: AuthResponse | null;
  error: AuthError | null;
}>;
```

**Parameters:**
- `redirectPath` (optional) - Path to redirect after authentication (default: `/turismo?onboarding=1`)

**Example:**

```typescript
const { data, error } = await signInWithGoogle('/perfil');
if (error) {
  console.error('Sign in failed:', error);
}
```

### signOut()

Sign out the current user.

```typescript
function signOut(): Promise<{
  error: AuthError | null;
}>;
```

**Example:**

```typescript
const { error } = await signOut();
if (!error) {
  window.location.href = '/';
}
```

### getCurrentUser()

Get the currently authenticated user.

```typescript
function getCurrentUser(): Promise<User | null>;
```

**Example:**

```typescript
const user = await getCurrentUser();
if (user) {
  console.log('Logged in as:', user.email);
}
```
